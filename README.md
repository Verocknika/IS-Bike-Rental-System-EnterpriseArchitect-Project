### **Bike Rental Information System – Analysis & Design**

This repository contains the analysis and system design of a **Bike Rental Information System**, created as part of a university course focused on information system modeling and software architecture. The project was developed using *Enterprise Architect* and includes UML diagrams, system architecture, and complete documentation.

---

### **Project Overview**
The goal of this project was to design an information system for a bike rental service. The system covers customer management, bike inventory, reservations, payments, and operational workflows. The focus was on requirements analysis, modeling, and architectural design rather than implementation.

The project includes:
- Functional and non‑functional requirements  
- Use Case Model  
- Requirements diagram  
- Requirements realization (package: evidence of rentals)  
- Class model  
- System architecture  
- Exported UML diagrams (PNG/PDF)  
- Full project documentation (PDF)

---

### **Team Collaboration**
This project was created by a **team of four students**, collaborating remotely through **Microsoft Teams**. We met regularly online to discuss requirements, review diagrams, and coordinate our work throughout the project.

---

### **My Contribution**
I contributed primarily to the analytical parts of the project and completed several key deliverables independently. My responsibilities included:

- **Functional requirements** — full creation of the complete functional analysis; I prepared the entire set of functional requirements, while the rest of the team provided feedback.  
- **Non‑functional requirements** — full creation and documentation of all non‑functional requirements.  
- **Use Case Model** — complete creation of the use case model, including actors, use cases, and their relationships.  
- **Requirements diagram** — major contribution to the requirements diagram; I structured and refined most of the model, with the team adding comments and suggestions.  
- **Requirements realization (evidence of rentals)** — collaborative work where all team members contributed approximately equally.  
- **Class model** — significant involvement in creating and refining the class diagram, including attributes and relationships.  
- **Introduction and project context** — preparation of the introductory section describing the purpose, scope, and goals of the system.
